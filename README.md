# InkGrid

> 文章即知识库。一个知识型个人博客 + AI 问答平台 —— 访客读完文章还能向 AI 提问，AI 基于博主的文章回答并引用出处。

## 这是什么

InkGrid 把传统的"静态"个人博客变成"活"的：

- **博客即知识库**：博主发布的每一篇文章自动向量化入库，无需额外维护知识库。
- **访客可问**：完全公开，访客无需登录即可向 AI 提问，AI 基于文章内容作答并附上引用出处。
- **频道化场景**：博主的不同内容板块（个人经验、深圳政策研究等）成为"频道"，每个频道有独立的 AI 问答入口与预设人设。
- **语音通话**：问答支持豆包式实时语音，访客可解放双手与博主 AI 对话。

## 核心理念

> 传统的个人博客是"静"的——访客只能读。这个博客是"活"的——访客读完还能问，AI 替博主回答。

| 维度 | 传统博客 | InkGrid |
|------|---------|---------|
| 访客角色 | 只读 | 读 + 问 |
| 知识库 | 与博客分离 | 博客文章即知识库 |
| 问答对象 | 无 | 全网公开访客 |
| 场景扩展 | 加页面 | 加频道 |
| 模态 | 文字 | 文字 + 语音 |

## 信息架构

```
博客主站（公开阅读）
├── 文章（自动入库为知识库）
├── 频道（内容分组 + 独立问答）
│   ├── 个人经验（= 原 HR 助手，代答人设）
│   ├── 深圳政策（= 原政策查询，采集 + 录入）
│   └── 可扩展更多频道
└── AI 问答（贯穿全站）
    ├── 范围：全站 / 某频道 / 单篇文章
    └── 模式：文字 / 语音通话
```

## 技术栈概览

| 层 | 选型 |
|----|------|
| 前端 | Next.js 14 (App Router) + TypeScript + Tailwind + Ant Design + Zustand |
| 后端 | Python + FastAPI |

## 设计基调

采用 **Obsidian Spatial — Technical Minimalism** 设计系统：

- **纯黑底色**（`#121414`）最大化 OLED 对比
- **白色网格线** 6% 透明铺底，营造空间纵深
- **零圆角几何** 所有元素对齐 4px 基准
- **双字体系统** Geist（UI）+ JetBrains Mono（技术标签/代码）
- **层级靠边框与亮度差**，不用强阴影

详见 [plan/DESIGN.md](./plan/DESIGN.md)。
